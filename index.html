VSAG is a C++ library for approximate nearest-neighbor search. It ships production-ready HGraph, HNSW, DiskANN, IVF, Pyramid and SINDI indexes with SIMD-accelerated distance kernels, quantization, and a unified builder API — with first-class Python bindings via pyvsag.

Why VSAG

Engineered for production similarity search

SIMD-accelerated kernels

AVX2 / AVX-512 / NEON distance kernels, integer and float quantization (SQ4/SQ8, PQ, RaBitQ), and careful memory layout for hot paths.

A family of indexes

HGraph, HNSW, DiskANN, IVF, Pyramid, SINDI and brute force — one builder-style API, one serialization format, one set of ops.

Adaptive runtime

Index::Tune with the ELP optimizer searches the graph entry-point and beam parameters to keep recall and latency on target as workloads drift.

Flexible serialization

Serialize to BinarySet, streams, or user-supplied write callbacks. Zero-copy ReaderSet keeps cold indexes on disk and hot tiers in memory.

Range & hybrid search

Radius search, filtered search with user predicates, and a hybrid dense + sparse index for RAG and retrieval pipelines.

Pluggable resources

Bring your own Allocator and ThreadPool via Resource to integrate cleanly with the host database or service runtime.

Quickstart

Up and running in a few lines

Python (pyvsag)

pip install pyvsag
import pyvsag, numpy as np

index = pyvsag.Index("hgraph", """
{
  "dtype": "float32",
  "metric_type": "l2",
  "dim": 128,
  "index_param": { "base_quantization_type": "sq8" }
}""")

ids  = np.arange(10000, dtype=np.int64)
vecs = np.random.rand(10000, 128).astype("float32")
index.build(vectors=vecs, ids=ids, num_elements=10000, dim=128)

q = np.random.rand(128).astype("float32")
I, D = index.knn_search(vector=q, k=10, parameters='{"hgraph":{"ef_search":100}}')

C++

// link against libvsag, C++17
#include <vsag/vsag.h>

auto result = vsag::Factory::CreateIndex("hgraph", R"({
  "dtype": "float32", "metric_type": "l2", "dim": 128,
  "index_param": { "base_quantization_type": "sq8" }
})");
auto index = result.value();

auto dataset = vsag::Dataset::Make();
dataset->Dim(128)->NumElements(n)->Float32Vectors(data)->Ids(ids)->Owner(false);
index->Build(dataset);

auto q = vsag::Dataset::Make();
q->Dim(128)->NumElements(1)->Float32Vectors(query)->Owner(false);
auto r = index->KnnSearch(q, 10, R"({"hgraph":{"ef_search":100}})");

Indexes

Pick the right algorithm for your workload

Index Best for Memory Build Query
hgraph General-purpose, high recall, quantized graph Medium Fast Very fast
hnsw Classic hierarchical navigable small world High Medium Fast
diskann Billion-scale corpora on SSD Low (RAM) Medium Fast (I/O bound)
ivf High-throughput batch search, GPU-friendly training Low–Medium Fast Fast
pyramid Multi-tenant / partitioned corpora Medium Fast Fast
sindi Sparse vectors (BM25-style, learned sparse) Low Fast Very fast
brute_force Exact ground-truth baseline, small datasets High Instant Slow
